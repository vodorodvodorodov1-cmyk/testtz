1. Исполнительное резюме
Настоящий документ содержит полную техническую спецификацию для разработки MVP фитнес-приложения с интегрированным ИИ-тренером. Документ охватывает технологический стек, архитектуру, API, инфраструктуру, правовые требования, смету затрат и временные рамки.

Цели MVP
•	Персонализированные тренировки, генерируемые ИИ на основе данных пользователя
•	Голосовой и текстовый чат с ИИ-тренером в режиме реального времени
•	Трекинг прогресса: тренировки, питание, сон, вес
•	Интеграция с носимыми устройствами (Apple Watch, Google Fit, Garmin)
•	Монетизация через подписку (Freemium)

Область MVP намеренно ограничена. Цель — выпустить рабочий продукт за 4–5 месяцев, проверить гипотезы и привлечь первых платящих пользователей, а не строить полнофункциональную платформу.

3.2 ИИ-архитектура (RAG + Агенты)
ИИ-тренер реализован как конвейер: Retrieval-Augmented Generation поверх базы знаний + агентная система для планирования тренировок.

Компонент	Технология	Назначение
Embedding-модель	text-embedding-3-small (OpenAI)	Векторизация упражнений, планов, сообщений пользователя
Vector store	pgvector (PostgreSQL-расширение)	Хранение и поиск по сходству (cosine similarity)
LLM Router	LangChain Router + LiteLLM	Выбор модели: mini для чата, большая для генерации планов
Память агента	Mem0 / Custom summarizer	Долговременный контекст: цели, прогресс, предпочтения
RAG Pipeline	LangChain LCEL	Document retrieval → re-ranking → context → LLM
Голосовой ввод	Whisper API (OpenAI)	Транскрипция голосовых сообщений пользователя
Стриминг ответов	Server-Sent Events (SSE)	Вывод токенов ИИ в реальном времени (typing effect)
Мониторинг LLM	LangSmith / Langfuse	Трассировка, оценка качества, стоимость запросов

3.3 Безопасность данных здоровья
Слой	Меры защиты
Транспорт	TLS 1.3 везде; HSTS; Certificate pinning в мобильном приложении
Хранилище	Шифрование на уровне диска (AES-256); Шифрование отдельных полей здоровья (pgcrypto)
Аутентификация	JWT (access 15 мин) + Refresh токен (30 дней); MFA опционально; Biometrics на устройстве
Авторизация	RBAC: user, trainer, admin; Row-Level Security в PostgreSQL
Аудит	Неизменяемый лог всех операций с данными здоровья (append-only таблица)
Удаление данных	Right-to-be-forgotten: полное удаление в течение 30 дней по запросу
 
4. API-спецификация
4.1 Конвенции
•	Базовый URL: https://api.fitai.app/v1
•	Формат: JSON, UTF-8
•	Аутентификация: Bearer JWT в заголовке Authorization
•	Пагинация: cursor-based (cursor, limit=20, has_next)
•	Ошибки: { "error": { "code": "...", "message": "...", "details": {...} } }
•	Rate limit: 60 req/min для free, 300 req/min для premium
